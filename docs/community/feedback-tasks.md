# Feedback tasks

RuleOak Core is currently in a feedback-first contribution stage. Issues and Discussions are welcome. Pull requests are disabled while contribution governance and licensing terms are finalized.

The tasks below are good first ways to help without submitting code.

## 1. Test the quickstart

Run the first-user path on your platform:

```bash
npm install
npm run launch
```

Open an issue if any command fails, the output is unclear, or the next step is not obvious.

Copy-ready issue: [feedback-tasks/01-test-quickstart.md](feedback-tasks/01-test-quickstart.md)

## 2. Review the governed demos

Run:

```bash
npm run demo
npm run report:view
```

Open feedback if the policy decision, evidence record, approval boundary, or audit report is hard to understand.

Copy-ready issue: [feedback-tasks/02-review-governed-demos.md](feedback-tasks/02-review-governed-demos.md)

## 3. Review documentation clarity

Read the README, quickstart, sandbox foundation, and Python SDK bridge docs. Open an issue for unclear wording, missing context, broken links, or setup instructions that need screenshots or examples.

Copy-ready issue: [feedback-tasks/03-review-docs-clarity.md](feedback-tasks/03-review-docs-clarity.md)

## 4. Suggest integration patterns

Share how RuleOak should integrate with existing agent frameworks, local LLM workflows, Python tools, developer automation, or governance stacks.

Copy-ready issue: [feedback-tasks/04-suggest-integration-pattern.md](feedback-tasks/04-suggest-integration-pattern.md)

## 5. Challenge the governance model

Ask hard questions about policy, evidence, approval, audit, sandbox boundaries, and where human review should be required.

Copy-ready issue: [feedback-tasks/05-challenge-governance-model.md](feedback-tasks/05-challenge-governance-model.md)

## Security concerns

Do not open public issues for vulnerabilities. Use the process in `SECURITY.md`.
